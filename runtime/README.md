# loamss runtime

The Go runtime for Loamss. Single binary, OS-level daemon. See the [spec set in the repo root](../README.md) for the design; this directory holds the reference implementation.

## Status

**Pre-Phase-1 scaffolding.** Currently only `loamss version` works. Phase 1 components land progressively per [`ROADMAP.md`](../ROADMAP.md): config loading, lifecycle (`start`/`stop`/`status`), the storage adapter, the MCP surface, the audit log writer, the capsule host.

The point of this commit is to establish the project shape — module layout, build tooling, CI — so subsequent additions are normal incremental work rather than scaffolding.

## Requirements

- Go 1.22+
- `make` (any POSIX `make` will do; tested with GNU Make)
- `golangci-lint` for `make lint` (CI installs this automatically; `brew install golangci-lint` locally)
- `git` for version metadata baking

## Build

```bash
make build       # → bin/loamss
make test        # unit tests
make test-race   # tests with the race detector
make lint        # golangci-lint
make fmt         # gofmt -s
make vet         # go vet
make tidy        # go mod tidy
make install     # install to /usr/local/bin (override with PREFIX=...)
make clean
make help        # list all targets
```

Try it:

```bash
make build
./bin/loamss version
```

Expected output:

```
loamss <version>
  commit:     <short-sha>
  built:      <ISO timestamp>
  go version: go1.22.x
  platform:   darwin/arm64    # or whatever
```

## Conventions

Documented across the spec set (see [CLAUDE.md](../CLAUDE.md) in the repo root for the digest). The load-bearing ones for code:

- Errors are values; wrap with context: `fmt.Errorf("opening config: %w", err)`
- No panics outside `main`
- No global state in the runtime (one exception: the cobra command tree, which cobra requires)
- Small interfaces, prefer accepting them over concrete types
- No string-matching against specific capsule names, client names, or capability names — see [`extensibility.md`](../extensibility.md) for the full anti-pattern list
